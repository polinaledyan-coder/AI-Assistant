# Проект: Личный ИИ-ассистент для саморазвития и планирования

## 1. Концепция проекта

Целью проекта является создание интеллектуального **личного ассистента**, предназначенного для помощи конкретному человеку в повседневной деятельности, саморазвитии и управлении временем. В отличие от общих голосовых помощников, данный ассистент обладает глубоким пониманием личности пользователя, его целей и предпочтений благодаря кастомизированной базе знаний.

Ассистент решает три основные задачи: выступает в роли собеседника и мотиватора, предоставляет доступ к персонализированной информации и автоматизирует процесс планирования мероприятий (встреч, уроков, тренингов) с интеграцией в календарь.

---

## 2. Целевая аудитория

*   Люди, практикующие саморазвитие и нуждающиеся в дисциплинирующем напоминании.
*   Занятые профессионалы, которым требуется помощь в планировании личного времени.
*   Клиенты коучей, психологов или онлайн-школ (в качестве дополнительного сервиса).

---

## 3. Ключевой функционал (MVP)

Проект реализует три основных сценария взаимодействия:

### 3.1. Осмысленный диалог (Личный коуч)
Ассистент способен поддерживать беседу, учитывая контекст личности пользователя. Он не просто отвечает на запросы, а может задавать уточняющие вопросы, мотивировать, давать обратную связь, опираясь на заложенную в него модель мышления.

### 3.2. Ответы по базе знаний (Экспертная система)
В ассистента загружена база знаний, содержащая структурированную информацию (книги, методики, лекции, правила, цели пользователя).
*   *Пример:* Пользователь спрашивает: "Напомни основные принципы моей утренней рутины" или "Какой метод запоминания я практикую?".
*   Ассистент находит ответ в собственной базе знаний (RAG — Retrieval-Augmented Generation), а не в интернете, что делает ответы персонализированными.

### 3.3. Управление расписанием (Планировщик)
Критически важная функция для поддержания регулярности в саморазвитии.
1.  **Инициация:** Пользователь просит записать его на событие (например: "Запиши меня на занятие йогой завтра в 19:00").
2.  **Обработка:** Ассистент распознает сущности (название события, дату, время).
3.  **Интеграция:** Через API происходит создание события в **Google Календаре** пользователя с автоматическим добавлением напоминаний.

## 4. Технологический стек и архитектура решения

Проект построен по принципу "сборки" из готовых сервисов, что позволило быстро запустить продукт без написания сложного кода с нуля.

### 4.1. База знаний и разработка логики (Qwen)
*   **Задача:** Подготовка "мозга" ассистента.
*   **Решение:**
    *   С помощью языковой модели **Qwen** был разработан детальный **системный промпт**. В нем прописана личность ассистента: его стиль общения (дружелюбный, поддерживающий, но настойчивый), цели (помощь в саморазвитии) и границы компетенции.
    *   Текстовые материалы пользователя (конспекты книг, личные цели, распорядки дня) были структурированы и загружены в формате, пригодном для поиска. Qwen помогла оптимизировать эти тексты для лучшего понимания их нейросетью.

### 4.2. Платформа для развертывания (Suvvy.ai)
*   **Задача:** Создание интерфейса для общения, интеграция базы знаний и подключение внешних сервисов без программирования.
*   **Решение:**
    *   **Suvvy.ai** выступил в качестве основы (движка) для ассистента.
    *   **Подключение базы знаний:** В интерфейсе Suvvy был создан раздел "База знаний", куда загружены подготовленные с помощью Qwen документы. Теперь при вопросах пользователя AI ищет ответ именно в этих файлах.
    *   **Интеграция с Google Календарем:** В конструкторе Suvvy был настроен сценарий (Action). Когда пользователь просит записать его на встречу, AI распознает намерение (Intent) и передает параметры в Google Календарь через встроенный коннектор (или через Webhook).

### 4.3. Схема взаимодействия компонентов

1.  **Пользователь** пишет сообщение в чат (веб-версия или Telegram-бот, развернутый через Suvvy).
2.  **Платформа Suvvy** принимает запрос.
3.  **Обработка запроса:**
    *   Если вопрос по существу, Suvvy обращается к **Qwen** (или встроенной LLM) с инструкциями из системного промпта.
    *   Если вопрос касается фактов из документов, Suvvy ищет в загруженной **базе знаний** (Knowledge Base).
4.  **Выполнение действия:** Если запрос касается создания события, Suvvy отправляет команду в **Google Calendar API**.
5.  **Ответ:** Сгенерированный текст или подтверждение о создании события отправляется пользователю.

## 5. Пользовательский сценарий (Пример работы)

1.  **Утро:** Пользователь пишет ассистенту: "Привет, что у меня по плану на сегодня?"
2.  **Ассистент:** "Доброе утро! По твоей базе знаний, сегодня у тебя запланировано повторение материала по английскому. Также напоминаю, что вечером в 19:00 у тебя стоит встреча с наставником в календаре."
3.  **Пользователь:** "Да, запиши еще на получасовую медитацию перед встречей, в 18:00."
4.  **Ассистент:** "Хорошо. Создаю событие 'Медитация' в Google Календаре на сегодня, 18:00 - 18:30. Добавить напоминание за 10 минут?" (Пользователь подтверждает).
5.  **Пользователь:** "Кстати, какой метод дыхания я использую для снятия стресса перед важными встречами?"
6.  **Ассистент (ищет в Базе знаний):** "Согласно твоему конспекту книги 'Вдох', ты практикуешь метод квадратного дыхания: вдох-задержка-выдох-задержка по 4 секунды."

---

## 6. Заключение и дальнейшее развитие

Данный проект демонстрирует эффективный симбиоз современных AI-инструментов. Использование **Qwen** позволило качественно подготовить семантическое ядро и базу знаний, а платформа **Suvvy.ai** стала идеальным конструктором для сборки функционального интерфейса с интеграциями.

**Потенциал развития:**
*   Подключение голосового ввода/вывода.
*   Интеграция с Telegram/WhatsApp для более удобного доступа.
*   Добавление аналитики (отчеты о выполнении целей за неделю).
